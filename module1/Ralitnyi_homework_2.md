# Звіт про виконання завдань Linux №2

---

## Завдання 1. Ієрархія каталогів Linux

### 1.1 Кореневий каталог `/`

```bash
cd /
ls
```

**Результат:**
```
bin    boot   dev   etc   home  lib    lib64
media  mnt    opt   proc  root  run    sbin
srv    sys    tmp   usr   var
```

**Опис ключових каталогів:**

| Каталог | Призначення |
|---------|-------------|
| `/bin`  | Основні виконувані файли (команди для всіх користувачів) |
| `/etc`  | Конфігураційні файли системи |
| `/home` | Домашні каталоги користувачів |
| `/root` | Домашній каталог суперкористувача root |
| `/tmp`  | Тимчасові файли |
| `/usr`  | Програми та бібліотеки для користувачів |
| `/var`  | Змінні дані: логи, кеш, черги |
| `/proc` | Віртуальна файлова система процесів |
| `/dev`  | Файли пристроїв (диски, термінали тощо) |

---

### 1.2 Каталог `/etc`

```bash
cd /etc
ls
```

**Результат (фрагмент):**
```
adduser.conf   apt         bash.bashrc   ca-certificates
cron.d         dpkg        fstab         group
hostname       hosts       issue         passwd
profile        resolv.conf shadow        shells
ssl            sysctl.conf timezone      ...
```

**Пояснення:** `/etc` містить усі системні конфігураційні файли. Найважливіші:
- `passwd` — список користувачів системи
- `group` — групи користувачів
- `fstab` — таблиця монтування файлових систем
- `hosts` — локальна таблиця DNS-імен
- `resolv.conf` — налаштування DNS

## Завдання 2. Файли, каталоги та посилання

### 2.1 Створення каталогу у домашньому каталозі

```bash
mkdir -p ~/lab2
ls -la ~ | grep lab2
```

**Результат:**
```
drwxr-xr-x 2 root root 4096 Jun 22 19:48 lab2
```

---

### 2.2 Створення файлу всередині каталогу

```bash
echo "Це мій перший файл у lab2" > ~/lab2/file.txt
ls -la ~/lab2/
```

**Результат:**
```
total 12
drwxr-xr-x 2 root root 4096 Jun 22 19:48 .
drwx------ 6 root root 4096 Jun 22 19:48 ..
-rw-r--r-- 1 root root   42 Jun 22 19:48 file.txt
```

---

### 2.3 Копіювання файлу

```bash
cp ~/lab2/file.txt ~/lab2/file_copy.txt
ls -la ~/lab2/
```

**Результат:**
```
-rw-r--r-- 1 root root 42 Jun 22 19:48 file.txt
-rw-r--r-- 1 root root 42 Jun 22 19:48 file_copy.txt
```

---

### 2.4 Перейменування копії

```bash
mv ~/lab2/file_copy.txt ~/lab2/file_renamed.txt
ls -la ~/lab2/
```

**Результат:**
```
-rw-r--r-- 1 root root 42 Jun 22 19:48 file.txt
-rw-r--r-- 1 root root 42 Jun 22 19:48 file_renamed.txt
```

**Пояснення:** `mv` без зміни каталогу працює як перейменування.

---

### 2.5 Жорстке посилання (hard link)

```bash
ln ~/lab2/file.txt ~/lab2/file_hardlink.txt
ls -lai ~/lab2/
```

**Результат:**
```
573460 -rw-r--r-- 2 root root 42 Jun 22 19:48 file.txt
573460 -rw-r--r-- 2 root root 42 Jun 22 19:48 file_hardlink.txt
573461 -rw-r--r-- 1 root root 42 Jun 22 19:48 file_renamed.txt
```

**Пояснення:** `file.txt` і `file_hardlink.txt` мають **однаковий inode (573460)** — це означає, що вони вказують на одні й ті самі дані на диску. Лічильник посилань збільшився до `2`. Видалення одного файлу не видаляє дані.

---

### 2.6 Символічне посилання (symlink)

```bash
ln -s ~/lab2/file.txt ~/lab2/file_symlink.txt
ls -lai ~/lab2/
```

**Результат:**
```
573460 -rw-r--r-- 2 root root  42 Jun 22 19:48 file.txt
573460 -rw-r--r-- 2 root root  42 Jun 22 19:48 file_hardlink.txt
573461 -rw-r--r-- 1 root root  42 Jun 22 19:48 file_renamed.txt
573462 lrwxrwxrwx 1 root root  19 Jun 22 19:48 file_symlink.txt -> /root/lab2/file.txt
```

**Пояснення:** Символічне посилання (`l` на початку прав) — окремий файл зі своїм inode (573462), який містить **шлях** до оригіналу. Якщо оригінал видалити, посилання стане «зламаним».

**Різниця між hard link і symlink:**

| Параметр | Hard link | Symbolic link |
|----------|-----------|---------------|
| Inode | Однаковий з оригіналом | Власний |
| Тип | `–` (звичайний файл) | `l` (symlink) |
| Крос-розділи | Ні | Так |
| При видаленні оригіналу | Дані зберігаються | Посилання «зламане» |

**Результат:**

## Завдання 3. Права доступу

### 3.1 Перегляд поточних прав файлу

```bash
ls -l ~/lab2/file.txt
```

**Результат:**
```
-rw-r--r-- 2 root root 42 Jun 22 19:48 /root/lab2/file.txt
```

**Розшифровка `-rw-r--r--`:**

```
- rw- r-- r--
│  │   │   └── Інші: тільки читання (r)
│  │   └────── Група: тільки читання (r)
│  └────────── Власник: читання + запис (rw)
└───────────── Тип: звичайний файл (-)
```

---

### 3.2 Надати файлу права тільки на читання

```bash
chmod 444 ~/lab2/file.txt
ls -l ~/lab2/file.txt
```

**Результат:**
```
-r--r--r-- 2 root root 42 Jun 22 19:48 /root/lab2/file.txt
```

**Пояснення:** `444` = `r--r--r--` — лише читання для всіх. Числовий запис: 4 (читання) для власника, групи та інших.

---

### 3.3 Надати власнику право на запис

```bash
chmod u+w ~/lab2/file.txt
ls -l ~/lab2/file.txt
```

**Результат:**
```
-rw-r--r-- 2 root root 42 Jun 22 19:48 /root/lab2/file.txt
```

**Пояснення:** `u+w` — додає (`+`) право запису (`w`) тільки власнику (`u`). Символьний запис зручніший для точкових змін.

**Таблиця символів chmod:**

| Символ | Означення |
|--------|-----------|
| `u` | user (власник) |
| `g` | group (група) |
| `o` | others (інші) |
| `a` | all (всі) |
| `+` | додати право |
| `-` | забрати право |
| `=` | встановити точно |
| `r` | читання (4) |
| `w` | запис (2) |
| `x` | виконання (1) |

---

### 3.4 Переглянути значення `umask`

```bash
umask
```

**Результат:**
```
0022
```

**Пояснення:** `umask` — маска, яка **віднімається** від максимальних прав при створенні нових файлів:
- Максимум для файлу: `666` → `666 - 022 = 644` (`-rw-r--r--`)
- Максимум для каталогу: `777` → `777 - 022 = 755` (`drwxr-xr-x`)

---

### 3.5 Встановити нове значення `umask`

```bash
umask 022
umask
```

**Результат:**
```
0022
```

**Поширені значення umask:**

| umask | Файли | Каталоги | Рівень безпеки |
|-------|-------|----------|----------------|
| `022` | `644` | `755` | Стандартний |
| `027` | `640` | `750` | Підвищений |
| `077` | `600` | `700` | Максимальний (тільки власник) |

## Завдання 4. Користувачі

### 4.1 Створення нового користувача

```bash
useradd -m -s /bin/bash trainee
```

**Параметри:**
- `-m` — створити домашній каталог `/home/trainee`
- `-s /bin/bash` — встановити bash як командну оболонку

**Перевірка:**
```bash
id trainee
```

**Результат:**
```
uid=1001(trainee) gid=1002(trainee) groups=1002(trainee)
```

---

### 4.2 Додавання до sudo-групи

```bash
usermod -aG sudo trainee
```

**Параметри:**
- `-a` — append (додати, не замінити)
- `-G sudo` — додати до групи `sudo`

**Перевірка:**
```bash
groups trainee
```

**Очікуваний результат:**
```
trainee : trainee sudo
```

---

### 4.3 Перевірка існування користувача

```bash
grep "trainee" /etc/passwd
```

**Результат:**
```
trainee:x:1001:1002::/home/trainee:/bin/bash
```

**Розшифровка запису `/etc/passwd`:**

```
trainee : x    : 1001 : 1002 : : /home/trainee : /bin/bash
   │      │       │      │    │       │               │
   │      │       │      │    │       │               └── Оболонка
   │      │       │      │    │       └── Домашній каталог
   │      │       │      │    └── Коментар (порожній)
   │      │       │      └── GID (ID групи)
   │      │       └── UID (ID користувача)
   │      └── Пароль (x = зберігається в /etc/shadow)
   └── Ім'я користувача
```
