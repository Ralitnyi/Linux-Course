# Звіт про виконання завдань Linux (Теми 1–5)

---

## Завдання 1. Базові команди

### 1. Вміст домашнього каталогу

```bash
ls ~
```

**Результат:** Домашній каталог відображено. У даному середовищі він порожній (щойно створений).

---

### 2. Перехід у каталог Downloads і його вміст

```bash
mkdir -p ~/Downloads
cd ~/Downloads
ls -la
```

**Результат:**
```
total 8
drwxr-xr-x 2 root root 4096 Jun 22 19:23 .
drwx------ 5 root root 4096 Jun 22 19:23 ..
```

---

### 3. Створення порожнього файлу **без** `touch`

```bash
> ~/Downloads/myfile.txt
```

**Пояснення:** Оператор `>` перенаправляє порожній вивід у файл, що створює його з нульовим розміром — без використання команди `touch`.

**Перевірка:**
```
-rw-r--r-- 1 root root 0 Jun 22 19:23 /root/Downloads/myfile.txt
```

---

### 4. Перегляд вмісту файлу

```bash
cat ~/Downloads/myfile.txt
```

**Результат:** Файл порожній (вміст відсутній — ми його щойно створили).

---

### 5. Перехід у домашній каталог **абсолютним** шляхом

```bash
cd /home/claude
pwd
```

**Результат:**
```
/home/claude
```

**Пояснення:** Абсолютний шлях починається з `/` — кореневого каталогу. Він не залежить від поточного місця знаходження.

---

### 6. Перехід у домашній каталог **відносним** шляхом

```bash
cd ~/Downloads
cd ..
pwd
```

**Результат:**
```
/root
```

**Пояснення:** `cd ..` — це відносний шлях, що означає «перейти на рівень вище». Він залежить від поточного каталогу.

## Завдання 2. Робота з документацією

### Виконані команди

```bash
man ls
help cd
man cat
man man
cp --help
mv --help
```

---

### Відповіді на питання

#### 1. Який ключ `ls` показує приховані файли?

```bash
ls --help | grep "\-a"
```

**Відповідь:** Ключ `-a` (або `--all`).

```
-a, --all    do not ignore entries starting with .
```

Приховані файли в Linux починаються з крапки (`.`). Команда `ls -a` показує всі файли, включно з прихованими (`.bashrc`, `.profile` тощо).

---

#### 2. Який ключ у `cat` нумерує рядки?

```bash
cat --help | grep "\-n"
```

**Відповідь:** Ключ `-n` (або `--number`).

```
-n, --number    number all output lines
```

Приклад: `cat -n файл.txt` виводить кожен рядок з його номером.

---

#### 3. Чим відрізняються `man` і `--help`?

| Параметр | `man` | `--help` |
|---|---|---|
| **Повнота** | Повна документація з усіма розділами | Короткий довідник |
| **Навігація** | Інтерактивна (пошук, прокрутка) | Прокрутка терміналу |
| **Де зберігається** | Окремі man-сторінки на диску | Вбудовано в саму програму |
| **Вивід** | Відкривається у `less` (пейджер) | Виводиться у `stdout` |
| **Зручність** | Детальніший, для глибокого вивчення | Швидше, для нагадування синтаксису |

**Команда `help`** (вбудована в bash) — окремий механізм для вбудованих команд bash (`cd`, `echo`, `pwd`), оскільки вони не є окремими програмами і не мають man-сторінок у стандартному вигляді.

## Завдання 3. Міні-сценарій

**Мета сценарію:** Навчитись переміщуватись по файловій системі, створювати файли та переглядати документацію.

### Команди сценарію

```bash
OAOAOA# Крок 1 — Перейти в каталог /tmp
cd /tmp
pwd
```
**Результат:** `/tmp`

---

```bash
# Крок 2 — Створити файл у поточному каталозі (без touch)
> scenario_file.txt
echo "Це тестовий файл" > scenario_file.txt
ls -l scenario_file.txt
```
**Результат:**
```
-rw-r--r-- 1 root root 31 Jun 22 19:23 scenario_file.txt
```

---

```bash
# Крок 3 — Переглянути список файлів у каталозі
ls -la /tmp/
```
**Результат (фрагмент):**
```
-rw-r--r-- 1 root root 31 Jun 22 19:23 scenario_file.txt
```

---

```bash
# Крок 4 — Перейти в інше місце (абсолютним шляхом)
cd /etc
pwd
```
**Результат:** `/etc`

---

```bash
# Крок 5 — Повернутись назад у /tmp
cd /tmp
pwd
```
**Результат:** `/tmp`

---

```bash
# Крок 6 — Переглянути вміст створеного файлу
cat scenario_file.txt
```
**Результат:**
```
Це тестовий файл
```

---

```bash
# Крок 7 — Переглянути документацію команди ls
ls --help
```
**Результат (перші рядки):**
```
Usage: ls [OPTION]... [FILE]...
List information about the FILEs (the current directory by default).
  -a, --all    do not ignore entries starting with .
  -n, --number number all output lines
  ...
```

---

### Підсумок сценарію

| Крок | Команда | Дія |
|------|---------|-----|
| 1 | `cd /tmp` | Перейти у каталог |
| 2 | `> scenario_file.txt` | Створити файл без touch |
| 3 | `ls -la` | Переглянути список файлів |
| 4 | `cd /etc` | Перейти в інше місце |
| 5 | `cd /tmp` | Повернутись назад |
| 6 | `cat scenario_file.txt` | Переглянути вміст файлу |
| 7 | `ls --help` | Документація команди |

## Використані команди (теми 1–5)

- `ls` — перегляд вмісту каталогу
- `cd` — зміна поточного каталогу
- `pwd` — вивести поточний шлях
- `cat` — вивести вміст файлу
- `>` — перенаправлення виводу (створення файлу)
- `echo` — вивід тексту
- `man` / `--help` — перегляд документації
- `mkdir` — створення каталогу
