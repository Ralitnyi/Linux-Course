# Звіт про виконання завдань Linux — Процеси та моніторинг ресурсів

---

## Завдання 1. Огляд активних процесів

### 1.1 Виведення всіх процесів через `ps`

```bash
ps aux
```

**Пояснення ключів:**

| Ключ | Значення |
|------|----------|
| `a` | показати процеси всіх користувачів |
| `u` | розширений формат (CPU, MEM, VSZ тощо) |
| `x` | включити процеси без терміналу |

**Результат (перші рядки):**
```
USER       PID %CPU %MEM    VSZ   RSS TTY      STAT START   TIME COMMAND
root         1 12.5  0.1  15140  5356 ?        Sl   08:05   0:00 /process_api ...
root         2  0.0  0.0      0     0 ?        S    08:05   0:00 [kthreadd]
root         3  0.0  0.0      0     0 ?        S    08:05   0:00 [pool_workqueue_release]
root         9  0.0  0.0      0     0 ?        I    08:05   0:00 [kworker/0:0-events]
root        11  0.4  0.0      0     0 ?        I    08:05   0:00 [kworker/0:1-events]
...
```

**Розшифровка стовпців:**

| Стовпець | Значення |
|----------|----------|
| `PID` | Ідентифікатор процесу |
| `%CPU` | Використання процесора |
| `%MEM` | Використання пам'яті |
| `VSZ` | Віртуальна пам'ять (КБ) |
| `RSS` | Реальна пам'ять у RAM (КБ) |
| `STAT` | Стан процесу (R=running, S=sleeping, Z=zombie) |
| `TTY` | Термінал (`?` = без терміналу) |

---

### 1.2 Моніторинг `top` / `htop` та процес з найбільшим RAM

**Інтерактивний запуск `top`:**
```bash
top
```

Корисні клавіші в `top`:
- `M` — сортування за пам'яттю
- `P` — сортування за CPU
- `q` — вийти

**Снімок процесів, відсортованих за `%MEM`:**
```bash
ps aux --sort=-%mem | head -10
```

**Результат:**
```
USER       PID %CPU %MEM    VSZ    RSS TTY   STAT START  TIME COMMAND
root       491 63.2  0.8 1955924 35484 ?     Sl   08:05  0:00 /opt/rclone/rclone-filestore multimount ...
root         1 12.4  0.1   15144  5428 ?     Sl   08:05  0:00 /process_api --firecracker-init ...
root       521  0.0  0.1    7900  4188 ?     R    08:05  0:00 ps aux --sort=-%mem
...
```

**Висновок:** Найбільше RAM споживає процес `rclone-filestore` (PID 491) — **0.8% / ~35 МБ (RSS)**.

**Якщо встановлений `htop`:**
```bash
htop
```
`htop` зручніший — показує кольорові смужки CPU/RAM, підтримує мишу, F6 для сортування.

---

### 1.3 PID поточної оболонки

```bash
echo "PID поточного bash: $$"
ps -p $$ -o pid,ppid,cmd
```

**Результат:**
```
PID поточного bash: 518

  PID  PPID CMD
  518     1 /bin/sh
```

**Пояснення:** `$$` — спеціальна змінна bash, що містить PID поточного процесу. `PPID` — батьківський процес (тут PID 1 = init/systemd).

## Завдання 2. Робота у фоні та керування процесами

### 2.1 Запуск команди у фоновому режимі

```bash
sleep 1000 &
```

**Результат:**
```
[1] 525
Запущено sleep 1000, PID: 525
```

**Пояснення:** Символ `&` в кінці команди запускає її у фоновому режимі. Shell виводить `[номер_завдання] PID`.

---

### 2.2 Список фонових завдань

```bash
jobs -l
```

**Результат:**
```
[1] + 525 Running    sleep 1000
```

**Пояснення стовпців:**
- `[1]` — номер завдання в поточній сесії
- `+` — поточне (активне) фонове завдання
- `525` — PID процесу
- `Running` — стан процесу
- `sleep 1000` — команда

---

### 2.3 Повернення процесу на передній план

```bash
fg %1
```

**Результат:** Процес `sleep 1000` повернено на передній план. Термінал блокується до завершення процесу. Щоб знову відправити у фон: `Ctrl+Z` (зупинити) → `bg %1` (продовжити у фоні).

**Корисні команди керування:**

| Команда | Дія |
|---------|-----|
| `fg %N` | Вивести завдання N на передній план |
| `bg %N` | Продовжити завдання N у фоні |
| `Ctrl+Z` | Зупинити (призупинити) поточний процес |
| `Ctrl+C` | Завершити поточний процес |

---

### 2.4 Зупинка та примусове завершення через `kill`

**М'яке завершення (SIGTERM — сигнал 15):**
```bash
kill 525
```

**Результат:**
```
Процес 525 більше не існує
```

**Примусове завершення (SIGKILL — сигнал 9):**
```bash
sleep 2000 &        # Запускаємо новий процес (PID: 528)
kill -9 528
```

**Результат:**
```
Процес 528 знищено через SIGKILL (-9)
```

**Основні сигнали `kill`:**

| Сигнал | Номер | Дія |
|--------|-------|-----|
| `SIGTERM` | 15 | М'яке завершення (за замовчуванням) — процес може «прибратися» |
| `SIGKILL` | 9 | Примусове знищення — ядро завершує негайно |
| `SIGSTOP` | 19 | Призупинити процес |
| `SIGCONT` | 18 | Продовжити призупинений процес |
| `SIGHUP` | 1 | Перезапустити конфігурацію (hangup) |

> **Правило:** Завжди спочатку `kill PID` (SIGTERM). Якщо не допомагає — `kill -9 PID` (SIGKILL).

---

### 2.5 `nohup` — процес продовжує роботу після закриття терміналу

```bash
nohup sleep 300 > /tmp/nohup_test.out 2>&1 &
```

**Результат:**
```
nohup sleep 300 запущено, PID: 531

  PID  PPID STAT CMD
  531   524 S    sleep 300
```

**Пояснення:**
- `nohup` — «No Hang Up», ігнорує сигнал SIGHUP, що надсилається при закритті терміналу
- `> /tmp/nohup_test.out` — перенаправляє stdout у файл (замість стандартного `nohup.out`)
- `2>&1` — перенаправляє stderr туди ж
- `&` — запускає у фоні

**Перевірити запущений nohup-процес можна навіть після нового логіну:**
```bash
ps aux | grep "sleep 300"
```

## Завдання 3. Пріоритети та обмеження

### 3.1 Запуск команди з підвищеним значенням `nice`

```bash
nice -n 15 sleep 500 &
```

**Результат:**
```
Запущено sleep 500 з nice=15, PID: 535

  PID  NI CMD
  535  15 sleep 500
```

**Шкала пріоритетів `nice`:**

```
-20  ─────────────────────────────  +19
 ↑                                   ↑
Найвищий                         Найнижчий
пріоритет                        пріоритет
(тільки root)              (доступний всім)
```

**Пояснення:**
- Значення **`nice`** — від -20 (найвищий пріоритет) до +19 (найнижчий)
- Звичайний користувач може підвищити nice тільки до +19 (знизити пріоритет)
- Лише **root** може встановити від'ємне значення (підвищити пріоритет)
- За замовчуванням: `nice = 0`

---

### 3.2 Зміна пріоритету вже запущеного процесу (`renice`)

```bash
renice -n 10 -p 535
```

**Результат:**
```
535 (process ID) old priority 15, new priority 10

  PID  NI CMD
  535  10 sleep 500
```

**Синтаксис `renice`:**
```bash
renice -n ЗНАЧЕННЯ -p PID          # за PID
renice -n ЗНАЧЕННЯ -u USER         # всі процеси користувача
renice -n ЗНАЧЕННЯ -g GROUP        # всі процеси групи
```

---

### 3.3 Обмеження ресурсів через `ulimit`

```bash
ulimit -a
```

**Результат:**
```
time(seconds)         unlimited    # максимальний час CPU
file(blocks)          unlimited    # максимальний розмір файлу
data(kbytes)          unlimited    # максимальний розмір даних
stack(kbytes)         8192         # розмір стеку (8 МБ)
coredump(blocks)      0            # розмір core dump (вимкнено)
memory(kbytes)        unlimited    # максимум пам'яті
locked memory(kbytes) 8192         # заблокована пам'ять
process               15955        # максимальна кількість процесів
nofiles               1024         # максимум відкритих файлів
vmemory(kbytes)       unlimited    # віртуальна пам'ять
locks                 unlimited    # файлові блокування
rtprio                0            # реальний пріоритет
```

**Корисні ключі `ulimit`:**

| Команда | Значення |
|---------|----------|
| `ulimit -a` | Показати всі обмеження |
| `ulimit -n` | Ліміт відкритих файлів |
| `ulimit -u` | Максимум процесів |
| `ulimit -s` | Розмір стеку |
| `ulimit -n 4096` | Встановити ліміт файлів на 4096 |

> **Примітка:** Зміни `ulimit` діють тільки для поточної сесії. Для постійних змін — редагуйте `/etc/security/limits.conf`.

## Завдання 4. Моніторинг ресурсів

### 4.1 Використання дискового простору

```bash
df -h
```

**Результат:**
```
Filesystem       Size  Used  Avail  Use%  Mounted on
tmpfs            2.0G     0  2.0G     0%  /dev/shm
tmpfs            2.0G     0  2.0G     0%  /sys/fs/cgroup
/dev/vda         252G  8.6G   10G    47%  /
rclone-filestore 1.0P     0  1.0P     0%  /mnt/user-data/outputs
rclone-filestore 1.0P     0  1.0P     0%  /mnt/user-data/uploads
```

**Пояснення:**
- `-h` — human-readable (МБ/ГБ/ТБ замість байтів)
- **Основний розділ `/dev/vda`:** 252 ГБ загалом, 8.6 ГБ зайнято, 10 ГБ вільно (47%)
- `tmpfs` — тимчасові файлові системи у RAM

**Інші корисні варіанти:**
```bash
df -h /          # тільки кореневий розділ
du -sh ~/lab2    # розмір конкретного каталогу
du -sh /* 2>/dev/null | sort -h  # найбільші каталоги
```

---

### 4.2 Оперативна пам'ять

```bash
free -h
```

**Результат:**
```
               total    used    free   shared  buff/cache  available
Mem:           3.9Gi   209Mi   3.8Gi    4.2Mi       72Mi      3.7Gi
Swap:             0B      0B      0B
```

**Розшифровка:**

| Поле | Значення |
|------|----------|
| `total` | Загальний обсяг RAM: **3.9 ГБ** |
| `used` | Зайнято: **209 МБ** |
| `free` | Абсолютно вільно: **3.8 ГБ** |
| `buff/cache` | Використовується ядром для кешу: **72 МБ** |
| `available` | Реально доступно для нових програм: **3.7 ГБ** |
| `Swap` | Розділ підкачки: **не налаштований** |

> **Важливо:** `available` ≠ `free`. Пам'ять у `buff/cache` може бути звільнена при потребі, тому реально доступно **3.7 ГБ**, а не лише 3.8 ГБ вільних.

**Детальна інформація з `/proc/meminfo`:**
```bash
grep -E "MemTotal|MemFree|MemAvailable|Cached|Buffers" /proc/meminfo
```

**Результат:**
```
MemTotal:     4093928 kB    ← ~4.0 ГБ загалом
MemFree:      3964340 kB    ← ~3.8 ГБ вільно
MemAvailable: 3879524 kB    ← ~3.7 ГБ доступно
Buffers:        11568 kB    ← ~11 МБ буферів
Cached:         58044 kB    ← ~57 МБ кешу
SwapCached:         0 kB    ← swap не використовується
```
